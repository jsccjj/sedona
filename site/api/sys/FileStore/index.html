    FileStore¶

    Doc Home > API Index > sys > FileStore

    Inheritance¶

    sys::Obj
  sys::Virtual
    sys::FileStore
    public class FileStore

    FileStore is used to implement the backing storage of File based I/O.
    Typically each storage device such as static RAM or EEPROM implements a subclass of FileStore and declares a singleton instance, inserting it into the FileStore static linked list at runtime. Filenames are mapped to a FileStore instance via 'FileStore.lookup', which steps through the static linked list and calls accept() on each one, returning the first instance that returns true.

    insert¶

    public static bool insert(FileStore fsObj)

    Insert fsObj at end of FileStore list.

    This function should be called by the constructor of an instance of a FileStore subclass, to insert itself into the list.

    Always returns true. No protection against adding an instance multiple times.

    lookup¶

    public static FileStore lookup(Str filename)

    Return first FileStore instance from list that returns true from accept(). Returns FileStore.sys if no instance returns true.
